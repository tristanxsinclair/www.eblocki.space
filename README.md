# Eblocki

**Convert ambition into measurable proof.**

Eblocki is a behavioural performance operating system designed to turn goals, study plans, sales improvement, sport development, content building, and identity work into concrete evidence.

It is not a generic productivity app.  
It is not a motivational dashboard.  
It is not a basic chatbot.

Eblocki is built around one operating principle:

> Proof beats intention.

The app helps users move through a clear performance loop:

Check-in → Mode Detection → State Diagnosis → Coaching Response → Proof Contract → Proof Artifact → Evidence Score → Feedback → Next Upgrade

---

## Core Identity

Eblocki is built around the identity:

> “I am an analytical operator who converts ambition into measurable proof.”

The system is designed for users who want direct, structured execution across academic performance, work performance, personal discipline, sport, content, and long-term capability building.

---

## Product Philosophy

Eblocki is based on a proof-first performance model:

- Proof beats intention.
- Systems beat motivation.
- Depth beats aesthetic productivity.
- Feedback beats ego protection.
- Confidence is the receipt, not the starting point.
- Success is the disciplined conversion of chosen values into repeated, feedback-corrected behaviour.

The app does not ask, “What do you feel like doing?”

It asks:

> “What proof artifact will confirm completion?”

---

### AI Coach

The AI Coach is designed to act as a behavioural diagnostic engine.

It takes a user check-in or task and returns:

- Detected mode
- Detected behavioural state
- Strategic coaching response
- Next controllable action
- Proof Contract when a serious action is required

Default response structure:

1. Bottom Line Up Front
2. Analysis
3. Actionable System
4. HD/Elite Upgrade

---

### Proof Contracts

A Proof Contract converts a serious next action into a required artifact.

Example:

Task:  
Prepare for LAWS1005 statutory interpretation.

Proof Contract:  
Submit one 250–400 word statutory interpretation IRAC using text, context, purpose, competing construction, and conclusion.

A Proof Contract answers the question:

> “What proof artifact will confirm completion?”

---

### Proof Artifacts

Proof Artifacts are completed evidence items.

Examples include:

- Law answer
- Psychology paragraph
- Sales reflection
- Product comparison
- Soccer training log
- Scooter/content progress
- Eblocki build step
- Financial tracker
- Written self-audit
- Code commit note
- Resume or cover letter asset

Artifacts are scored by evidence strength.

---

### Evidence Scoring

Eblocki scores proof from 1–10.

| Score | Evidence Strength | Meaning |
|---|---|---|
| 1–3 | Weak | Vague, shallow, no applied evidence |
| 4–6 | Moderate | Concrete artifact exists but limited depth |
| 7–8 | Strong | Applied skill, clear structure, useful feedback |
| 9–10 | Elite | Artifact, application, feedback, correction, and next upgrade |

The aim is not to feel productive.  
The aim is to generate evidence.

## Behavioural States

Eblocki also detects behavioural state when relevant.

| State | Meaning | Response |
|---|---|---|
| `locked_in` | High clarity and execution | Protect momentum and ship proof |
| `avoidant` | Circling the task | Reduce scope and define first artifact |
| `overloaded` | Too many demands | Triage to one prime objective |
| `low_energy` | Low capacity | Use a five-minute entry version |
| `hype_drift` | Big ideas, weak execution | Force a shippable artifact |
| `recovery` | Rest is needed | Controlled recovery with light proof |
| `momentum` | Execution has started | Continue, do not redesign |
| `scattered` | Fragmented attention | One objective, one artifact, one timer |
| `academic_displacement` | Studying around the marked task | Return to answer-ready work |
| `strategic_build` | Long-term building mode | Ship a small feature and document the decision |
