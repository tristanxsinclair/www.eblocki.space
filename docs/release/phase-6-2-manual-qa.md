# Phase 6.2 Manual QA

Status: checklist added. Do not mark manual QA complete until these checks are run against the intended environment.

## Proof page - product-system proof

1. Open `/proof`.
2. Choose artifact type `product system review`.
3. Confirm `Product System Review Standard` appears before submission.
4. Confirm Required Evidence includes:
   - specific product issue identified
   - evidence from actual output/screen
   - corrected logic proposed
   - implementation path stated
   - measurable test or acceptance criterion
   - next upgrade defined
5. Confirm law-answer criteria do not appear:
   - AGLC4
   - binding/persuasive sources
   - text-context-purpose
   - statutory interpretation
6. Submit a product-system proof without implementation/test proof.
7. Confirm verdict missing standard is product-system relevant and mentions implementation/test proof.
8. Confirm identity escalation is blocked until implementation or external test evidence exists.

## Proof page - law source-bank proof

1. Open `/proof`.
2. Choose artifact type `source-bank entries` or link a source-bank Proof Contract.
3. Confirm `Law Source Bank Standard` appears before submission.
4. Confirm Required Evidence includes:
   - source name
   - jurisdiction
   - authority level
   - current version checked
   - key provision/material
   - key rule
   - unit relevance
   - assessment use
   - limitation/counterargument
   - confidence rating
5. Confirm IRAC is not required before at least one authority/source-bank entry exists.
6. Confirm identity escalation says source-bank proof requires later applied use such as an issue matrix, paragraph, or problem answer.

## Proof page - IRAC proof

1. Open `/proof`.
2. Choose artifact type `IRAC paragraph`.
3. Confirm `Law IRAC Standard` appears when the selected mode/domain is law or a law contract is linked.
4. Confirm Required Evidence includes issue, rule, authority, application, and conclusion.
5. Confirm source-bank-only fields such as current version checked and confidence rating are not required for the IRAC artifact.

## Proof contract alignment

1. Link a pending Proof Contract.
2. Confirm Contract Alignment is visible before submission.
3. Confirm aligned contracts say Proof Action and Proof Contract require the same artifact type.
4. Confirm mismatched action/contract states warn and fallback to one visible artifact with one evidence standard.

## Dashboard

1. Open `/dashboard`.
2. Confirm one Today Command is visible.
3. Confirm one proof contract is shown.
4. Confirm selected standard appears inside the proof contract summary.
5. Confirm no competing proof artifact appears unless a multi-artifact sprint is explicitly active.

## Release evidence

Manual QA is complete only after screenshots or notes record the route/standard/contract result for product-system, source-bank, and IRAC proof previews.
